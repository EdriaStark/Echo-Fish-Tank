# Writing DNA

把一组文章整理成可复用的写作画像。上传文本，查看表达规律，再导出可直接交给 AI 的写作提示。

在线体验：[edriastark.github.io/Echo-Fish-Tank](https://edriastark.github.io/Echo-Fish-Tank/)

## 它解决什么

写作风格通常散落在几十篇文章里。Writing DNA 将这些文章留在浏览器中，归纳可检查的语言特征：词汇、句子节奏、语气、结构、主题和可复用提示。

这不是云端写作助手，也不会代替编辑判断。当前版本使用本地统计和明确的启发式规则；结果会标注样本强度，并可查看支撑摘要的原始片段。

## 核心体验

- 拖入 `.md`、`.txt` 或 ZIP 文件夹。
- 自动跳过相同内容，避免重复样本影响结果。
- 将文章保存在本机浏览器的 IndexedDB 中；下次打开仍可继续。
- 从示例报告开始，无需先准备文章。
- 复制 AI-ready prompt 或导出 Markdown 报告。
- 所有文本在浏览器本地处理，不会上传到服务器。

## 使用方式

1. 打开网页，选择「查看示例」或拖入文章。
2. 上传至少 5 篇文章，获得更有参考价值的画像。
3. 展开「查看样本片段」，核对结果是否符合你的判断。
4. 复制提示词，在你选择的 AI 工具中继续写作。

建议使用 20 篇或更多完整文章。样本越多，结论越稳定。

## 技术说明

这是一个零构建步骤的静态应用，适合 GitHub Pages：

- 原生 HTML、CSS、JavaScript
- 按需加载 ZIP 解析和示例数据
- IndexedDB 本地持久化
- 无分析追踪、无服务端、无账户系统

## 本地预览与发布

直接用静态文件服务器预览根目录即可。发布时，将根目录推送到 GitHub 默认分支，在仓库 **Settings → Pages** 中选择 **Deploy from a branch** 与 `/(root)`。

## 发布前的仓库边界

该仓库目前还包含 `socialpeta-clean/`，它是与 Writing DNA 无关的独立项目。若要把本仓库用于 Product Hunt、GitHub Showcase 或作品集，请在公开发布前将该目录迁移到单独仓库（或从本仓库移除）。这项操作会改变仓库历史与公开内容，因此需要仓库所有者明确决定后再执行。
